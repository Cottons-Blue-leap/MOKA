**MOKA 작업 운용 구조** — 2026-03-26 합의

**Why:** 한 에이전트가 기획/아트/프로그래밍/QA 전부 맡으면 맥락이 흐려짐. 세션 단위로 집중 영역을 정하고 체크리스트를 처리하는 방식으로 효율화.

**How to apply:** 세션 시작 시 작업 영역과 체크리스트를 확인하고, 해당 영역에 집중. subagent는 병렬 실무에 활용.

## 작업 영역

### 1. 기획 (Planning)
- 기능 설계, 사용자 플로우, 경쟁 분석
- Notion 문서 연동
- 우선순위 판단

### 2. 프로그래밍 (Development)
- 기능 구현, 버그 수정, 리팩토링
- 빌드/배포

### 3. QA (Quality Assurance)
- 기능 점검, 레이아웃 검증, 다국어 확인
- 실기기 테스트 피드백 반영

### 4. 아트/UX (Art & UX)
- UI 디자인 가이드, 애니메이션, 테마
- ComfyUI 연동 (컨셉 아트)

## 세션 운용 방식
1. 세션 시작 → 메모리에서 현재 프로젝트 상태 + 미완료 체크리스트 확인
2. 코튼과 작업 영역 합의
3. 체크리스트 기반으로 작업 진행, subagent 병렬 활용
4. 세션 종료 전 → 완료/미완료 항목 메모리 업데이트
