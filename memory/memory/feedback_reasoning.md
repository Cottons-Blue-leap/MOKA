---
name: Reasoning quality in recommendations
description: When recommending option X over Y, provide reasons unique to X — not reasons that apply equally to all options
type: feedback
---

비교 추천 시, 해당 선택지만의 **고유한 강점**을 근거로 제시할 것. 모든 선택지에 동일하게 적용되는 장점은 근거가 아님.

**Why:** 네이밍 체계 A/B/C 추천 시, C의 근거로 제시한 3가지가 A, B에도 전부 해당됨을 사용자가 지적 (2026-03-23). "C가 좋다"가 아니라 "A, B, C 다 좋다"라고 말한 셈. 이는 추천이 아닌 나열이며, 사용자의 의사결정을 돕지 못함.

**How to apply:**
1. 선택지 비교 시, 각 선택지의 **차별점**을 먼저 정리
2. 추천 근거는 "이것은 X만 가능하고 Y/Z는 불가능하거나 약한 것"으로 구성
3. 만약 고유 강점을 찾기 어렵다면, 솔직하게 "세 가지 다 비슷한 수준이고, 취향 차이"라고 말할 것
4. 확신 없는 추천보다 솔직한 "잘 모르겠다"가 낫다
