## 보고서 작성
시각화와 리스크 분석을 반드시 포함할 것.
- 차트, 테이블, KPI 카드 등 시각화 요소 필수
- 리스크/위험 요소가 있다면 별도 섹션으로 포함
- **Why:** 텍스트만 나열된 보고서는 빠른 이해가 어렵고, 긍정적 면만 서술하면 올바른 경영 판단 불가.

## PPT/문서 작성
**1. 폰트** — 문서 목적에 맞는 폰트 선택. 기본 시스템 폰트(맑은 고딕 등)에 의존하지 말 것.
- 예: 기업 보고서 → 세련된 비즈니스 폰트, 캐주얼 자료 → 가독성 폰트

**2. 편집 편의성** — 인간이 수정하기 쉬운 구조.
- 표는 반드시 실제 테이블(table) 객체로 작성 (텍스트박스 나열 금지)
- 그래프/차트에서 글자 겹침 없도록 충분한 여백
- **Why:** 최종 품질은 인간의 미세 조정을 거쳐야 완성됨.

## 한글 품질
Notion 등에 한글 기록 시 자모 조합 오타에 주의할 것.
- 과거 사례: 뱅드림→뼉드림, 글자 겹침→글자 격침
- **Why:** 사용자가 직접 수정해야 하는 번거로움. 기록 신뢰도에 영향.
- **How to apply:** 고유명사, 전문용어는 특히 한 번 더 확인.
